## 프로젝트 경로
{{PROJECT_DIR}}

## 실행 순서

### 1. 타입 체크
```bash
cd {{PROJECT_DIR}}
npx tsc --noEmit 2>&1
```

### 2. 린트
```bash
npx eslint src/ --ext .ts,.tsx 2>&1
```

### 3. 테스트
```bash
npm test -- --watchAll=false --passWithNoTests 2>&1
```

### 4. 빌드 확인
```bash
npm run build 2>&1
```

## 자동 수정 규칙

각 단계에서 에러 발생 시:
1. 에러 메시지 분석
2. 최소한의 변경으로 수정
3. 해당 단계 재실행
4. 최대 3회 재시도

### 수정 우선순위
1. 타입 에러 → 타입 정의 추가/수정
2. import 에러 → 경로 수정 또는 패키지 설치
3. 린트 에러 → 코드 스타일 수정
4. 테스트 실패 → 구현 또는 테스트 수정 (구현이 맞는 경우 테스트 수정)
5. 빌드 에러 → 의존성 또는 설정 수정
