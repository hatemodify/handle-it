## 활용 스킬 (Skill 도구로 호출)
아래 스킬은 Skill 도구를 사용해 직접 호출하세요. 작업 품질이 크게 향상됩니다.

- **`/prd-check`**: PRD 초안 작성 후 반드시 호출. 누락 섹션, 모호한 기능 정의, 측정 불가 목표를 자동 점검합니다.
  → 호출: `Skill(skill: "prd-check")`
- **`/flow-audit`**: 사용자 플로우 작성 후 호출. 진입점 → 핵심 액션 → 완료/이탈 경로 커버리지, 엣지 케이스(에러, 빈 상태, 첫 사용) 정의 여부를 점검합니다.
  → 호출: `Skill(skill: "flow-audit")`

### 스킬 활용 순서
1. PRD 초안 작성
2. `/prd-check` 호출 → 피드백 반영
3. `/flow-audit` 호출 → 플로우 보완
4. 최종 PRD 저장

## 작성 형식

### 1. 개요
- 한 줄 설명
- 핵심 문제와 해결책

### 2. 타겟 사용자
- 주요 사용자 페르소나 (1-2개)

### 3. 핵심 기능 (우선순위 순)
각 기능마다:
- 기능명
- 설명 (1-2줄)
- 우선순위: P0(MVP 필수) / P1(중요) / P2(보완)
- 성공 지표 (측정 가능한 메트릭)

### 4. 사용자 플로우 (핵심 3가지)
각 플로우: 시작 → 단계 → 완료
- 정상 경로 (happy path)
- 에러/예외 경로 (edge case)
- 빈 상태 (empty state) 처리

### 5. MVP 범위
- 3주 안에 완성할 수 있는 최소 범위
- 포함 / 제외 항목 명시

### 6. 비기능 요구사항
- 성능 목표
- 보안 고려사항
- 플랫폼 (iOS/Android/Web)

### 7. PRD 자체 검증 (prd-check)
작성 완료 후 아래 항목을 점검하고 결과를 PRD 하단에 포함:
- [ ] 모든 P0 기능에 성공 지표가 있는가
- [ ] 사용자 플로우에 엣지 케이스가 정의되어 있는가
- [ ] MVP 범위가 3주 내 실현 가능한 수준인가
- [ ] 비기능 요구사항에 구체적 수치가 있는가
